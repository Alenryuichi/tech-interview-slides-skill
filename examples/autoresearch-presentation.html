MLP & FORWARD PASS

MLP 与前向传播

MLP 使用 ReLU² 激活函数，前向传播引入 per-layer residual lambda 和 x0 shortcut 实现训练稳定性。

MLP

class MLP(nn.Module):
    def forward(self, x):
        x = self.c_fc(x)      # 4×expand
        x = F.relu(x).square() # ReLU²
        x = self.c_proj(x)    # project back
        return x

Block Residual

# Per-layer residual + x0 skip
x = λ_resid[i] * x + λ_x0[i] * x0
ve = value_embeds[i](idx)
x = block(x, ve, cos_sin, ws)

# Logit soft-capping
logits = 15 * tanh(logits/15)

ReLU²

ReLU(x)² 产生更稀疏激活，改善梯度信号

Residual Lambda

可学习的 per-layer 缩放因子，稳定深层训练

Logit Softcap

tanh(z/15)×15 防止 logit 爆炸，稳定 loss

设计哲学

稳定优先 — λ_resid 初始化 1.0，λ_x0 初始化 0.1，softcap = 15，每个设计选择都为训练稳定服务

# 时间驱动的 LR 调度
progress = total_training_time / TIME_BUDGET  # 0→1

def get_lr_multiplier(progress):
    if progress < WARMUP_RATIO:       # 线性预热 (默认 0%)
        return progress / WARMUP_RATIO
    elif progress < 1.0 - WARMDOWN_RATIO:  # 恒定 LR
        return 1.0
    else:                              # 线性冷却 (50%)
        cooldown = (1.0 - progress) / WARMDOWN_RATIO
        return cooldown + (1 - cooldown) * FINAL_LR_FRAC

🔬 项目哲学 — 深度解析

📌 核心概念

autoresearch 的核心思想来自 Andrej Karpathy 的一个设想：给 AI Agent 一个真实的 ML 训练环境，让它自主做研究。不是模拟，不是 benchmark，而是真正的从零训练 LLM。

  • 自主研究 — Agent 独立提出假设、实现代码修改、运行实验、分析结果
  • 固定预算 — 每次实验精确 5 分钟 (wall clock training time)，排除编译/startup 时间
  • 进化压力 — 只有改善 val_bpb 的修改被保留，形成自然的"适者生存"

🏗️ 范式对比

传统 ML 研究
  • 人类手动设计实验、调参、分析
  • 受限于工作时间 (8h/天)
  • 实验间等待时间长
autoresearch 范式
  • Agent 24/7 自主实验
  • ~12 实验/小时，一夜 ~100 实验
  • 严格的 keep/discard 协议，无人工干预

⚙️ Simplicity Criterion

All else being equal, simpler is better. A 0.001 val_bpb improvement that adds 20 lines of hacky code? Probably not worth it. A 0.001 val_bpb improvement from deleting code? Definitely keep. An improvement of ~0 but much simpler code? Keep.

Evolutionary SearchFixed-Budget EvaluationSingle-Metric OptimizationMinimal Surface Area

核心思路：极简约束产生最大自由度——固定时间、固定指标、固定评估，其余一切由 Agent 决定。

🧠 GPT 模型 — 深度解析

📌 核心概念

从 nanochat 精简而来的 GPT 实现，保留了最有效的现代技巧：Value Embedding (ResFormer)、per-layer residual lambda、logit soft-capping。

🏗️ 模型组件

GPTConfig — Depth-Driven Sizing
  • base_dim = DEPTH × ASPECT_RATIO (默认 8 × 64 = 512)
  • model_dim = round_up(base_dim, HEAD_DIM) → 对齐到 128
  • num_heads = model_dim / HEAD_DIM → 自动推导
  • Agent 只需调 DEPTH 即可缩放整个模型
Residual Lambda + x0 Shortcut
  • x = λ_resid[i] * x + λ_x0[i] * x0 (可学习的 per-layer 标量)
  • λ_resid 初始化 1.0，λ_x0 初始化 0.1
  • x0 是 embedding 后的原始表示，提供"直达底层"的梯度路径
Logit Soft-Capping
  • logits = 15 * tanh(logits / 15)
  • 将 logit 范围限制在 [-15, 15]，防止 softmax 数值溢出
  • Gemma 2 引入的技术

⚙️ Weight Initialization

# Embedding: normal(0, 1) # lm_head: normal(0, 0.001) — 近零初始化 # Q/K/V: uniform(-s, s) where s = √3 * n_embd^(-0.5) # c_proj, mlp.c_proj: zeros — 残差分支初始为恒等 # VE gate: zeros → sigmoid(0) = 0.5, scaled by 2 → 1.0 (neutral)

🔗 相关设计模式

Dataclass ConfigDepth-Driven ScalingResidual ShortcutSoft-Capping

核心思路：所有超参数都由 DEPTH 派生，Agent 获得一个"单旋钮"接口来控制模型复杂度。

👁️ 注意力机制 — 深度解析

📌 核心概念

CausalSelfAttention 融合了 5 项现代注意力技术：RoPE 位置编码、QK-Norm、Value Embedding、Flash Attention 3、滑动窗口。

🏗️ 技术细节

Rotary Position Embedding (RoPE)
  • 预计算 cos/sin 缓存，seq_len × 10 长度余量
  • apply_rotary_emb: 将 x 分为两半，旋转乘法
  • 优点：相对位置感知 + 零额外参数 + 可外推
Value Embedding (ResFormer)
  • 交替层注入：has_ve(layer_idx, n_layer) → 每隔一层
  • Input-dependent gate: 2 * sigmoid(ve_gate(x[:, :, :32]))
  • Gate 初始化 0 → sigmoid(0) = 0.5 → 2×0.5 = 1.0 (neutral start)
  • 仅用前 32 维 (ve_gate_channels) 作为 gate 输入 → 参数高效
Sliding Window Attention
  • SSSL 模式: S=T/2 短窗口, L=T 全窗口
  • 每 4 层循环 SSSL: [1024, 1024, 1024, 2048]
  • 最后一层强制全局: window_sizes[-1] = (long_window, 0)
  • 短窗口减少计算量，长窗口保证全局信息流

⚙️ QK-Norm 实现

q, k = apply_rotary_emb(q, cos, sin), apply_rotary_emb(k, cos, sin) q, k = norm(q), norm(k) # RMS norm after RoPE # Flash Attention 3 call y = fa3.flash_attn_func(q, k, v, causal=True, window_size=window_size)

QK-Norm (RoPE 后 RMS Norm) 防止注意力分数过大，配合 FA3 的高效实现。Hopper GPU 使用 varunneal/flash-attention-3，非 Hopper 回退到 kernels-community/flash-attn3。

🔗 相关设计模式

RoPEQK-NormValue ResidualSliding WindowFlash Attention 3

核心思路：最大信息流 + 最小内存 — RoPE 零参数位置编码，VE 增强表达力，滑动窗口节省 O(T²) 中的 T。

⚡ MLP & Forward Pass — 深度解析

📌 核心概念

MLP 使用 ReLU² 而非 GELU/SiLU，前向传播通过 residual lambda 和 x0 shortcut 实现稳定的深层训练。

🏗️ ReLU² 激活

为什么用 ReLU² 而不是 GELU/SiLU?
  • F.relu(x).square() — 两次运算，比 GELU 更快
  • 产生更稀疏的激活模式 (负值归零 + 小值平方后更小)
  • 更清晰的梯度信号：导数 = 2·ReLU(x) (非零部分线性)
  • 在小模型规模下，ReLU² 与 SiLU 性能接近且更高效

⚙️ Forward Pass 完整流程

x = self.transformer.wte(idx) # token embedding x = norm(x) # 初始 RMS norm x0 = x # 保存原始表示 for i, block in enumerate(self.transformer.h): x = self.resid_lambdas[i] * x + self.x0_lambdas[i] * x0 ve = self.value_embeds[str(i)](idx) if str(i) in self.value_embeds else None x = block(x, ve, cos_sin, self.window_sizes[i]) x = norm(x) # 最终 RMS norm logits = self.lm_head(x).float() # float32 for precision logits = 15 * torch.tanh(logits / 15) # soft-capping

📊 FLOPs 估算

估算公式: 6 × (non-embedding params) + Σ(12 × H × D_head × effective_seq)，其中 effective_seq 取决于滑动窗口大小。

🔗 相关设计模式

ReLU²x0 ShortcutPer-Layer LambdaSoft-Capping

核心思路：每个组件都服务于训练稳定性——c_proj 零初始化让残差分支从恒等开始，λ_x0=0.1 提供弱但持续的底层梯度路径。

🔧 MuonAdamW 优化器 — 深度解析

📌 核心概念

MuonAdamW 是一个混合优化器：Muon 处理 2D 矩阵参数 (Q/K/V/projection)，AdamW 处理其余参数。两者通过统一的 step() 接口协作。

🏗️ Muon 四步流水线

1. Nesterov Momentum
  • 动量从 0.85 线性增到 0.95 (300 步预热)
  • g = grad.lerp_(momentum_buffer, momentum)
2. Polar Express 正交化
  • 将梯度矩阵正交化，消除参数间的冗余更新
  • 使用 5 次迭代的矩阵多项式近似: X = a*X + X @ (b*A + c*A²)
  • 系数硬编码在 polar_express_coeffs (5 组 float)
3. NorMuon 方差缩减
  • EMA 跟踪方差: second_momentum_buffer.lerp_(v_mean, 1-β₂)
  • 归一化更新幅度，使不同层的有效 LR 一致
4. Cautious Weight Decay
  • mask = (g * params) >= 0
  • 只对梯度和参数同号的维度施加 decay
  • 避免与梯度方向"打架"

⚙️ 参数分组策略

param_groups = [ (lm_head, 'adamw', lr=0.004), # unembedding (wte, 'adamw', lr=0.6), # token embedding (value_embeds, 'adamw', lr=0.6), # value embeddings (resid_λ, 'adamw', lr=0.005), # residual scalars (x0_λ, 'adamw', lr=0.5), # x0 scalars (matrix_*, 'muon', lr=0.04), # 2D weights (by shape) ] # LR ∝ 1/√(d_model/768) — 维度缩放

核心思路：参数的几何形状决定最优更新策略——2D 矩阵从正交化中受益，而 1D/embedding 更适合传统自适应方法。

🔄 训练循环 — 深度解析

📌 核心概念

训练循环以挂钟时间 (而非 step 数) 作为进度基准。前 10 步为 torch.compile 预热，不计入 TIME_BUDGET。

🏗️ 时间驱动调度

LR Schedule: Trapezoidal
  • Warmup (默认 0%): progress / WARMUP_RATIO
  • Constant (前 50%): LR = initial_lr
  • Warmdown (后 50%): 线性降至 FINAL_LR_FRAC × initial_lr
  • 所有 param_group 共享同一 multiplier
Muon Momentum Schedule
  • 前 300 步: 0.85 → 0.95 线性增长
  • 动量预热避免早期振荡
Weight Decay Schedule
  • WD × (1 - progress) — 线性衰减到零
  • 训练后期减少正则化，让模型更自由地拟合

⚙️ 关键实现细节

# Gradient accumulation tokens_per_fwdbwd = DEVICE_BATCH_SIZE * MAX_SEQ_LEN # 128 × 2048 = 262144 grad_accum_steps = TOTAL_BATCH_SIZE // tokens_per_fwdbwd # 524288 / 262144 = 2 # GC management: Python GC causes ~500ms stalls if step == 0: gc.collect(); gc.freeze(); gc.disable() elif (step + 1) % 5000 == 0: gc.collect() # EMA loss for logging (not used for decisions) smooth_train_loss = 0.9 * smooth_train_loss + 0.1 * train_loss_f

📊 数据管线 — 深度解析

📌 核心概念

数据管线分为一次性准备 (download + tokenizer) 和运行时 (dataloader + evaluation) 两个阶段，全部封装在 prepare.py 中作为不可变基础设施。

🏗️ Tokenizer 训练

rustbpe + tiktoken 双层架构
  • 训练: rustbpe (Rust 实现，快速 BPE 训练)
  • 推理: tiktoken (OpenAI 的高性能编码器)
  • Vocab size: 8192 (小而高效，适合小模型)
  • Split pattern: GPT-4 风格，但 \p{N}{1,2} 替代 {1,3}
  • 4 个 special tokens: <|reserved_0|> .. <|reserved_3|>

⚙️ Best-Fit Packing Dataloader

def make_dataloader(tokenizer, B, T, split): # 每行容量: T + 1 (用于 input/target shift) # 每行以 BOS 开头 # Packing 策略: # 1. 从 buffer 中找最大的能完整放入的文档 # 2. 如果没有文档能放入 → 裁剪最短文档填满剩余空间 # 结果: 100% 利用率, 零 padding # Memory: # row_buffer [B, T+1] — CPU 拼装 # cpu_buffer [2*B*T] — pinned memory # gpu_buffer [2*B*T] — CUDA # non_blocking copy: cpu → gpu

📊 BPB 评估

  • BPB = Bits Per Byte = total_nats / (ln2 × total_bytes)
  • token_bytes lookup: 每个 token 编码后的 UTF-8 字节数
  • Special tokens (byte_length=0) 被排除
  • 固定 EVAL_TOKENS = 20M tokens
  • 与 vocab size 无关 → 架构变化可公平对比

🔗 相关设计模式

Two-Phase PipelineBest-Fit Bin PackingPinned MemoryVocab-Invariant Metric

核心思路：零浪费 (100% utilization) + 公平评估 (BPB 不依赖 vocab size) = Agent 可以自由改变词汇表大小而不影响实验可比性。

🤖 Agent 集成 — 深度解析

📌 核心概念

program.md 本质上是一个超轻量级的 "skill" 或 "system prompt"，定义了 Agent 的行为协议。它不是代码，而是自然语言规则。

🏗️ 协议结构

Setup Phase
  • 与人类协商 run tag (如 mar5)
  • 创建分支: git checkout -b autoresearch/<tag>
  • 读取所有源文件建立上下文
  • 验证数据存在，初始化 results.tsv
Experiment Loop (LOOP FOREVER)
  • 修改 train.py → git commit → uv run train.py > run.log 2>&1
  • grep 解析结果 → 记录到 results.tsv (不 commit)
  • 改善 → 保留 commit; 未改善 → git reset
  • 崩溃 → 尝试修复，修不好就跳过
Constraints
  • 只能修改 train.py
  • 不能安装新包
  • 不能修改评估函数
  • 超时 (>10 min) → kill → 视为失败

⚙️ Logging Format

# results.tsv (tab-separated) commit val_bpb memory_gb status description a1b2c3d 0.997900 44.0 keep baseline b2c3d4e 0.993200 44.2 keep increase LR to 0.04 c3d4e5f 1.005000 44.0 discard switch to GeLU activation d4e5f6g 0.000000 0.0 crash double model width (OOM)

🎯 设计决策 — 深度解析

📌 决策 1: 固定时间 vs 固定 Step 数

选择: 固定时间 (5 分钟)
  • 实验可直接对比，无论 Agent 怎么改 batch size/model size
  • 自动适应不同硬件：H100 跑更多 step，小卡跑更少但同样 5 分钟
  • Agent 的优化目标变成"在固定时间内找最优模型"
代价: 跨平台不可比
  • H100 上的 baseline 和 RTX 3090 上的 baseline 数值不同
  • 但这是有意的：autoresearch 为你的硬件找最优模型

📌 决策 2: 单文件可编辑 vs 完全开放

限制 Agent 只改 train.py 看似限制了能力，但实际上：

  • 防止 Agent 修改评估函数"作弊"
  • 固定数据管线确保实验可重现
  • diff 可审计：人类可以理解每次修改的意图

📌 决策 3: BPB vs Perplexity

  • BPB: nats→bits 转换，按 byte (非 token) 归一化
  • 改变 vocab size 不影响 BPB → 架构搜索更公平
  • 代价: 需要额外的 token_bytes 查找表，计算稍慢

📌 决策 4: 编译预热排除

  • 前 10 步 torch.compile JIT 编译，耗时远超正常 step
  • 不计入 TIME_BUDGET → 评估的是训练性能，不是编译性能

🔗 相关设计模式

Fixed-Budget DesignSandboxed MutationVocab-Invariant MetricWarm-Start Exclusion

核心思路：每个约束都是精心设计的围栏——给 Agent 最大自由度的同时防止常见陷阱 (作弊、不可比、编译干扰)。
